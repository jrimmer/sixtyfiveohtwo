# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in SixtyFiveOhTwo, please report it privately using GitHub's security advisory feature:

**[Report a vulnerability](https://github.com/jrimmer/sixtyfiveohtwo/security/advisories/new)**

Please do **not** open a public issue for security vulnerabilities.

## What to Include

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

## Response Timeline

- **Acknowledgment**: Within 48 hours
- **Initial assessment**: Within 1 week
- **Resolution target**: Depends on severity

## Scope

This policy applies to:
- The main application server
- All game implementations (Telengard, Sabotage, Proving Grounds)
- The deployed production instance

## Out of Scope

- Vulnerabilities in dependencies (report to upstream maintainers)
- Social engineering attacks
- Denial of service attacks
